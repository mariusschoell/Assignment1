Mobile applications

Web Applications

Web Applications are stored on a remote server and get delivered over the internet through a browser. They are not real apps, they are basically websites that may sometimes look and feel like native applications. Usually, they are written in HTML5. They provide interaction, navigation or customization capabilities and they’re completely delivered on the fly via the internet. Nowadays, the distinction between web apps and regular web pages is quite blurry.

Native Applications

Developed for the use on only a platform or device. It is a smartphone application that is coded in a specific programming language (f. e. Objective for iOS or Java for Android).
Native apps have a fast performance and a high degree of reliability. It is expensive to develop native apps because it is tied to only one type of operating system, which forces the developers to make duplications of the app that work on other platforms. A native app can only be “native” to one type of mobile operating systems (iOS, Android, Blackberry…).

Hybrid Applications

Hybrid apps are like native apps. Hybrid Apps run inside a native container and the browser’s engine renders locally. Hybrid apps are written with the same technology like websites and mobile web implementations. Sometimes, companies build hybrid apps as wrappers for an existing web page so they get a presence in app stores without a lot of effort. Hybrid apps allow cross-platform development which means, that code components can be reused on different mobile operating systems and that means a lower cost.

Mobile application platforms

According to Gartner Mobile Application Development Platforms (MADP) are “ provides tools, technologies, components and services that together constitute the critical elements of a platform for creating custom mobile apps”.
In the 2017 report OutSystems, Mendix, Oracle, Microsoft, Salesforce, Kony and Progress are stated as leaders. Therefore those can be seen as the most important ones.

The platforms available vary in the amount of coding that has to be done by the creator of a MADP.
Other sources recognize also recognize Appcelerator, Appzillon, Apple Xcode, Android Studio, Kinvey, Telerik and Red Hat Mobile Application Platform.

Sources: Gartner Reprint, TechTarget

Cross Platform Mobile Application Development?

This means, that the created applications are compatible with multiple mobile operating systems. They are not bound to only one.
Cross-platform mobile development can either involve a company developing the original app on a native platform (which could be iOS, Android...) or developing the original app in a singular environment for development that will then allow the app to be sent to many different native platforms.

Techniques to develop different types of mobile applications

Web applications are usually programmed in the programming language HTML5. Examples for frameworks are Meteor, Rails or Laravel.
Native applications are programmed in the language that is typical for the system they are going to run in. For Android it is Java or Objective and for iOS devices it is Swift. Other frameworks like React Native and Flutter allow the development of native apps for all platform that use other languages than the standard one.
Hybrid applications use a hybrid framework (f. e. Apache Cordova, Ionic or Appcelerator Titanium). The programming languages are HTML5, CSS and JavaScript.

Responsive Web Design (RWD) vs. Mobile First approach

Responsive Web Design (RWD)

Starting point is the view on the computer screen and only when that is finished, the mobile view get’s important. Responsive web design is a behaviour of the completed product.

Mobile First Approach

Mobile first means designing an online experience for mobile before designing it for the desktop. Starting point is the mobile view. Mobile first is a design process.
